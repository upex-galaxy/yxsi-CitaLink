# Fase 10: Exploratory Testing

## Propósito
Testing manual rápido para dar feedback ANTES de invertir en automation.

## Por qué manual antes que automatizado
- Feedback rápido (minutos vs horas)
- Encuentra bugs de UX que tests automatizados no ven
- Solo automatizas lo ya validado manualmente
- Shift-left = feedback lo más temprano posible

## Orden de ejecución
1. smoke-test.md (primero - validar deployment)
2. test-charter.md (crear charter de exploración)
3. session-notes.md (documentar sesión)
4. bug-report.md (reportar bugs encontrados)

## Output
- Smoke test pasando
- Sesiones de exploración documentadas
- Bugs reportados en Jira
- Funcionalidad validada manualmente
